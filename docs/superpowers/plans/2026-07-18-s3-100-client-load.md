# S3 100-client load test implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and run a production-safe, repeatable load test for 100 simultaneous lesson image uploads plus a 25-client multipart video phase.

**Architecture:** A Node runner calls the existing authenticated multipart lesson-upload API, records per-operation timings, samples health/metrics, verifies S3 objects, and deletes every completed object in `finally`. Pure helper functions hold profiles, thresholds, statistics, and stage planning so they can be developed test-first.

**Tech Stack:** Node.js 22 fetch, Vitest, TypeScript workspace, Hono API, AWS S3-compatible storage, Docker production monitoring.

## Global Constraints

- Production image phase is exactly 100 clients at 2 MiB each.
- Production video phase is exactly 25 clients at 24 MiB each.
- API RSS safety limit is 1.5 GB and upload HTTP error-rate limit is 1%.
- Production execution requires `CONFIRM_PRODUCTION_LOAD=YES`.
- No lesson database records are created.
- Cleanup runs after success, failure, and interruption.

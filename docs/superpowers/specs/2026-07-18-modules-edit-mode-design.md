# Режим редактирования модулей

## Цель

Сделать список модулей одинаковым для администратора и клиента по умолчанию. Административные действия появляются только после явного включения режима редактирования.

## Интерфейс

- В шапке раздела «Модули» для администратора с правом управления материалами показываются две кнопки: карандаш и плюс.
- Карандаш расположен слева от плюса, как в разделе оплаты.
- Карандаш работает как переключатель, имеет `aria-pressed` и визуально выделяется во включённом состоянии.
- Плюс продолжает открывать создание нового модуля независимо от режима редактирования.
- При выключенном режиме скрыты: сортировка модулей, редактирование модуля, добавление урока, сортировка и удаление карточек, удалённый контент.
- При включённом режиме все существующие административные действия возвращаются без изменения их бизнес-логики.
- Сворачивание и разворачивание модуля остаётся доступным всегда.
- Открытие урока и просмотр его содержимого остаются доступными всегда. Отдельная кнопка редактирования внутри экрана урока не меняется.

## Состояние и доступ

- Локальное состояние `isEditingModules` по умолчанию равно `false` при каждом новом открытии приложения.
- Переключатель доступен только при `canManageModules`.
- Потеря права управления материалами немедленно выключает режим редактирования.
- API, права, сортировка, удаление и структура данных не меняются.

## Проверка

- Unit-тест фиксирует наличие переключателя и условия видимости всех административных действий.
- Проверяется порядок кнопок: карандаш перед плюсом.
- Проверяется доступность: подпись, `aria-pressed`, активный класс.
- Проверяется отсутствие изменений для обычного пользователя.
- Выполняются тесты learning-раздела, TypeScript check, production build и мобильная визуальная проверка.

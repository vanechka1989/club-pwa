# Принудительный мобильный режим PWA

## Цель

На физическом телефоне интерфейс Club PWA всегда работает в мобильной компоновке, даже если пользователь открыл сайт или установил PWA при включённой браузерной настройке «Версия для ПК». На настоящем компьютере приложение предупреждает, что основной сценарий рассчитан на телефон, но не блокирует пользователя при возможной ошибке определения.

## Выбранный подход

Использовать классификацию по нескольким независимым сигналам, а не по ширине окна или User-Agent в отдельности. Уже существующая диагностика дополняется явным режимом устройства:

- `mobile` — физический телефон в обычном мобильном режиме;
- `mobile-desktop` — физический телефон, которому браузер назначил широкий desktop viewport или desktop User-Agent;
- `desktop` — настоящий компьютер;
- `unknown` — недостаточно сигналов для безопасного решения.

Сигналы классификации: `navigator.userAgentData.mobile` при наличии, User-Agent и platform, `maxTouchPoints`, coarse/fine pointer, физические размеры `screen`, DPR, viewport, ориентация и PWA `display-mode`. Ни один сигнал отдельно не считается доказательством.

## Поведение интерфейса

### Телефон и установленное PWA

- `mobile` и `mobile-desktop` всегда получают мобильную оболочку.
- Установленное на физическом телефоне PWA всегда использует нижнюю навигацию, мобильные размеры, мобильную типографику и компенсацию широкого desktop viewport.
- Desktop sidebar и desktop media-layout не включаются независимо от сообщаемой браузером ширины.
- Настройку «Версия для ПК» браузера приложение не меняет: оно только нейтрализует её влияние внутри Club PWA.

### Телефон в браузерном режиме ПК

Показывается заметное предупреждение:

> Включена версия сайта для компьютера  
> Для корректной работы отключите «Версия для ПК» в настройках браузера.

Доступна кнопка «Всё равно продолжить». После продолжения интерфейс всё равно принудительно остаётся мобильным.

### Настоящий компьютер

Показывается предупреждение:

> Приложение оптимизировано для телефона  
> Откройте его на смартфоне в мобильном режиме.

Предупреждение содержит QR-код текущего адреса и кнопку «Всё равно продолжить». После продолжения содержимое отображается в мобильной ширине по центру desktop-окна; это не затрагивает экран входа и доступность основных действий.

### Неопределённое устройство

Вход не блокируется. Используется существующая адаптивная компоновка без предупреждения. Это защищает планшеты, складные устройства, сенсорные ноутбуки и браузеры с ограниченной диагностикой от ошибочной блокировки.

## Компоненты и поток данных

1. Чистая функция классификации получает снимок сигналов устройства и возвращает режим вместе с причинами/уверенными признаками.
2. `deviceLayout` превращает режим и состояние установки PWA в эффективную компоновку `mobile` или `desktop`.
3. `App.vue` использует эффективную компоновку для выбора навигации и отображения предупреждения.
4. Предупреждение оформляется отдельным доступным компонентом поверх приложения с понятным заголовком, кнопкой продолжения и QR-кодом только на настоящем компьютере.
5. Решение «продолжить» сохраняется локально для текущего типа предупреждения, но при изменении классификации проверяется заново.
6. Диагностика устройства продолжает отправляться в существующем формате; новая классификация не является серверным механизмом безопасности.

## Безопасность и ошибки

- Определение устройства используется только для UX и никогда не выдаёт права или доступ.
- Жёсткой блокировки нет, поскольку браузерные сигналы можно скрыть или подменить.
- Ошибка чтения User-Agent Client Hints, localStorage, matchMedia или screen не ломает загрузку: классификация переходит в `unknown`.
- QR-код строится только из текущего адреса приложения без токенов, приватных параметров и данных сессии.

## Проверка

- Unit-тесты классификации: Android/iPhone mobile, Android desktop-site, iOS/iPad desktop-site, Windows/macOS/Linux desktop, touch laptop, unknown.
- Unit-тесты effective layout: установленное PWA на телефоне всегда mobile; desktop PWA остаётся desktop-warning; browser mobile-desktop получает mobile layout.
- Component-тесты предупреждений, кнопки продолжения и QR-кода.
- Проверка навигации: desktop sidebar не появляется на физическом телефоне при viewport 980/1024 px.
- Визуальная проверка 320, 390, 768, 1024 и 1440 px, включая standalone PWA и браузерный desktop mode.
- Полные `check`, `test`, `build`, обновление версии приложения и service worker перед публикацией.

## Не входит в задачу

- Изменение системной настройки браузера «Версия для ПК».
- Абсолютно достоверное аппаратное распознавание устройства.
- Серверная блокировка desktop-запросов.

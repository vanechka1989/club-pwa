# Полное закрытие обращения поддержки

## Цель

После подтверждения закрытия обращение должно завершаться как отдельная задача: экран обращения и его маршрут закрываются, пользователь возвращается к списку. В данных обращения должно сохраняться, кто и когда выполнил закрытие.

## Данные

В `support_tickets` добавляются nullable-поля `closed_at` и `closed_by_user_id`. При закрытии API атомарно устанавливает статус `closed`, время и текущего пользователя. Повторное чтение возвращает объект `closedBy` с публичными данными пользователя и `closedAt`.

Старые закрытые обращения сохраняют `null` в новых полях и продолжают отображаться без подписи о закрывшем.

## Интерфейс

После успешного ответа API обращение обновляется в локальном списке, затем очищается выбранное обращение и выполняется переход на `/support`. Это закрывает TaskScreen и его подложку.

В списке закрытых обращений и при повторном открытии показывается компактная подпись: `Закрыл(а): <имя> · <дата и время>`. Для английского интерфейса используется `Closed by`.

## Надёжность

Если API вернул ошибку, экран остаётся открытым и пользователь видит прежнее сообщение об ошибке. Миграция обратно совместима. Регрессионные тесты проверяют схему, API-контракт, сохранение закрывающего пользователя, закрытие маршрута и вывод подписи.

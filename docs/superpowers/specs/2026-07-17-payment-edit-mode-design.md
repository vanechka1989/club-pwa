# Режим редактирования оплаты

## Цель

Владелец по умолчанию видит раздел оплаты в клиентском виде. Административные элементы появляются только после явного включения режима редактирования.

## Интерфейс

- В шапке «Оплата» слева от существующего плюса находится кнопка с карандашом.
- Кнопка переключает локальный режим редактирования и получает заметное активное состояние.
- Плюс в шапке сохраняет существующее назначение — добавление платёжной системы.
- До включения режима скрыты карточка платёжной системы, плюс добавления тарифа, действия редактирования/скрытия/удаления, а также скрытые и архивные тарифы.
- Опубликованные тарифы и кнопки оплаты всегда выглядят так же, как у клиента.

## Ограничения

- Права доступа, API и платёжная логика не меняются.
- Переключатель доступен только владельцу, которому уже доступны административные действия оплаты.
- При уходе со страницы режим не сохраняется.

## Проверка

- Контрактный тест проверяет условное отображение каждого административного блока.
- Выполняются тесты web-приложения и production-сборка.

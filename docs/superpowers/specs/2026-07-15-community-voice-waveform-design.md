# Голосовые сообщения с живой волной

## Цель

Заменить нативный аудиоплеер и разрозненные панели записи одной компактной строкой в стиле Telegram-lite. Одинаковая интерактивная волна используется во время записи, перед отправкой и внутри отправленного сообщения.

## Состояния

- **Запись:** корзина, красная точка, таймер, живая волна с уровнями микрофона, кнопка остановки.
- **Предпрослушивание:** play/pause, интерактивная волна с заполнением прогресса, текущее и полное время, корзина, отправка.
- **Отправка:** элементы блокируются, кнопка отправки показывает вращающийся индикатор, запись сохраняется до успешного ответа API.
- **Сообщение:** play/pause, та же волна, перемотка нажатием или протягиванием, время. При ошибке play превращается в повтор.

## Поведение волны

Во время записи `AnalyserNode` Web Audio API снимает уровень сигнала и добавляет нормализованный столбец в ограниченный буфер. Если Web Audio API недоступен, запись продолжает работать с пульсирующим нейтральным рисунком.

После записи накопленные уровни остаются в предпрослушивании. В отправленном сообщении компонент строит волну из аудиофайла, если файл можно декодировать; иначе используется стабильный резервный рисунок. Проигранная часть окрашивается акцентом. Над волной располагается прозрачный range-контрол для доступной перемотки.

## Компоновка

Панель занимает одну строку над основным полем ввода, без вложенных карточек и текстовых кнопок. Иконки имеют доступные названия и сенсорную область не меньше 44 × 44 px. На ширине 320 px кнопки не переносятся и не создают горизонтальный скролл.

## Ограничения

- Существующие API, формат аудиофайлов и правила хранения не меняются.
- Лимит записи остаётся 5 минут.
- Ошибка отправки не удаляет записанный blob.
- Цвета берутся только из семантических переменных активной темы.
- Поддерживается русский и английский интерфейс.

## Проверка

Модульные тесты проверяют буфер уровней, формат времени, расчёт прогресса и перемотку. Компонентные тесты проверяют все состояния панели и отсутствие нативного `audio controls`. Визуальная проверка выполняется на 320, 390 и 768 px для записи, предпрослушивания и сообщения.

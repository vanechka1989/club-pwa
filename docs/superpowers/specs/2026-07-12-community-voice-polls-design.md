# Голосовые сообщения и опросы в чатах

## Цель

Добавить в существующие темы сообщества голосовые сообщения, изображения и опросы. Сохранить действующие ответы, реакции, закрепление, модерацию и автоматическое обновление чата. Добавить агрегированную статистику опросов в административный раздел.

## Голосовые сообщения

- В композере чата появляется кнопка микрофона.
- Нажатие запускает запись через `MediaRecorder`; во время записи видны длительность, остановка и отмена.
- После остановки пользователь получает предпрослушивание и кнопки «Отправить» и «Удалить».
- Максимальная длительность — 5 минут. Сервер проверяет длительность, MIME-тип и размер файла.
- Поддерживаются форматы, которые реально создают мобильные Chromium/WebKit: WebM/Opus, MP4/M4A/AAC и OGG. Сервер нормализует тип по содержимому и расширению.
- Файл хранится в S3 под отдельным префиксом `community/voice/`.
- Сообщение содержит ссылку на объект, MIME-тип, размер, длительность и дату автоматического удаления.
- Для автора с ролью `user` дата удаления равна времени отправки плюс 30 дней.
- Для автора с ролью `admin` или `owner` дата удаления отсутствует; файл хранится постоянно. Решение принимается по роли в момент отправки и не меняется при последующей смене роли.
- После удаления пользовательского файла сообщение остаётся, а вместо плеера отображается «Голосовое удалено по сроку хранения».
- Очистка запускается существующим серверным обслуживанием и удаляет сначала объект S3, затем очищает медиа-поля сообщения. Повторный запуск безопасен.
- В сообщении показывается компактный плеер: воспроизведение/пауза, прогресс, прошедшее и полное время. Одновременно играет только одно голосовое.

## Изображения

- Через кнопку `+` пользователь выбирает изображения из галереи или открывает камеру устройства.
- В одном сообщении разрешено от 1 до 10 изображений.
- До отправки показывается компактное предпрослушивание с возможностью удалить отдельный файл или отменить весь черновик.
- Клиент корректирует EXIF-ориентацию для предпросмотра, а сервер повторно проверяет файлы и оптимизирует их существующим конвейером изображений.
- Сервер принимает JPEG, PNG и WebP. HEIC/HEIF принимается только после успешного декодирования через `sharp`; иначе API возвращает понятную ошибку неподдерживаемого формата. Результат хранится в WebP с удалёнными метаданными.
- Каждое исходное изображение ограничено 15 МБ. Оптимизированный вариант имеет не более 1600 px по длинной стороне и сохраняется в WebP с качеством 82.
- Файлы хранятся в S3 под префиксом `community/images/`.
- Для автора с ролью `user` каждому изображению назначается удаление через 30 дней.
- Для автора с ролью `admin` или `owner` дата удаления отсутствует; изображения хранятся постоянно. Политика фиксируется по роли в момент отправки.
- В сообщении одно изображение показывается крупным превью, 2–4 — сеткой, 5–10 — сеткой с индикатором оставшегося количества.
- Нажатие открывает полноэкранный просмотр с перелистыванием, масштабированием и закрытием назад/крестиком.
- После очистки всех изображений сообщение остаётся с пометкой «Изображения удалены по сроку хранения». Если очищена только часть, оставшиеся изображения продолжают показываться.
- Ответы и закреплённая панель используют подпись «Изображение» или «N изображений».

## Опросы

- Опрос создаётся через кнопку `+` рядом с полем ввода.
- Форма содержит вопрос, 2–10 вариантов, переключатель «Можно выбрать несколько», переключатель «Анонимный опрос» и необязательную дату завершения.
- Опрос публикуется как отдельное сообщение текущей темы.
- До завершения пользователь может менять свой выбор. После завершения выбор заблокирован.
- Для одиночного опроса сохраняется один выбор, для множественного — набор выбранных вариантов.
- Результаты показывают количество и процент голосов по вариантам, общее число проголосовавших и статус завершения.
- В анонимном опросе клиент и администратор видят только агрегаты. В неанонимном администратор/владелец может открыть список проголосовавших по вариантам.
- Автор опроса с правами модератора может завершить его досрочно. Удаление сообщения использует существующую модерацию и не оставляет доступного для голосования опроса.
- Повторная отправка одного и того же выбора идемпотентна; уникальность обеспечивается парой `poll_id + user_id + option_id`.

## Модель данных и API

- В `club_chat_messages` добавляется тип сообщения: `text`, `voice`, `images` или `poll`.
- Медиа хранятся в общей таблице `club_message_attachments`: тип вложения, S3-ключ, MIME-тип, размер, длительность аудио, ширина и высота изображения, порядок и дата удаления. Голосовое сообщение содержит ровно одно аудиовложение, сообщение с изображениями — 1–10 изображений.
- Опросы хранятся в отдельных таблицах `club_polls`, `club_poll_options` и `club_poll_votes` с внешними ключами и каскадным удалением.
- Общий контракт `ClubMessage` получает дискриминированные nullable-поля `voice`, `images` и `poll`, сохраняя `body` для текста, системных сообщений и доступных превью ответов.
- Текстовое сообщение продолжает отправляться существующим JSON endpoint.
- Голосовое и изображения отправляются multipart endpoint текущей темы.
- Опрос создаётся JSON endpoint текущей темы; голосование и досрочное завершение имеют отдельные endpoint.
- Все новые endpoint используют текущую авторизацию, проверку членства, блокировку темы и мут пользователя.
- Ограничения проверяются на клиенте для удобства и повторно на сервере для безопасности.

## Ответы, закрепления и уведомления

- На голосовое, изображения и опрос можно отвечать, ставить реакции, закреплять и модерировать их как обычные сообщения.
- Превью ответа использует подписи «Голосовое сообщение», «Изображение», «N изображений» или вопрос опроса.
- Закреплённая панель использует такие же подписи и не пытается вывести пустой `body`.
- Уведомление об ответе содержит безопасное текстовое описание типа сообщения.

## Статистика

- В административной статистике появляется блок «Опросы».
- Общие показатели: создано опросов, активных, завершённых, уникальных участников, всего голосов и средний процент участия относительно пользователей, имевших доступ к теме.
- Таблица опросов показывает вопрос, тему, автора, даты, статус, количество участников и распределение по вариантам.
- Для неанонимных опросов доступна детализация участников; для анонимных сервер не возвращает личности даже владельцу.
- Статистика рассчитывается сервером из таблиц опросов и голосов, без изменения существующей статистики обучения и оплат.

## Ошибки и совместимость

- При запрете микрофона показывается понятное сообщение и сохраняется возможность отправлять текст и опросы.
- При обрыве загрузки черновик записи или выбранные изображения остаются доступными для повторной отправки до закрытия страницы.
- Неподдерживаемый браузер скрывает запись, но может воспроизводить поддерживаемые голосовые.
- Ошибки голосования возвращают актуальный опрос, чтобы интерфейс синхронизировался после гонки с завершением.
- Миграции совместимы со старыми текстовыми сообщениями: их тип по умолчанию `text`.

## Проверка

- Unit-тесты: нормализация аудиотипов и изображений, политика хранения ролей, расчёт срока удаления, идемпотентная очистка, порядок 1–10 изображений, валидация опроса, одиночное и множественное голосование, анонимность, завершение и статистические агрегаты.
- API-тесты: права доступа, мут, закрытая тема, multipart upload голосового и галереи, ограничения количества и размера, голосование и административная детализация.
- UI-тесты: состояния записи, предпрослушивание, отправка, истёкшее голосовое, выбор 1–10 изображений, галерея, полноэкранный просмотр, истёкшие изображения, создание опроса, выбор вариантов и отображение результатов.
- Мобильная проверка: 320, 360/390 и 768 px; Android Chromium и iOS/WebKit поведение разрешения микрофона; отсутствие горизонтального переполнения и зона нажатия не меньше 44 px.

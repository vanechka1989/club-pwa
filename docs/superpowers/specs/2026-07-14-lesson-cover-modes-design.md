# Режимы обложки урока

## Цель

Убрать белые углы у стандартных обложек и дать администратору явный выбор источника обложки урока: стандартная, собственный файл или первое визуальное вложение.

## Поведение

- `default`: карточка использует стандартную WebP-обложку текущей темы и формата модуля.
- `custom`: карточка использует загруженную администратором обложку. Поле загрузки показывается только в этом режиме.
- `first_material`: карточка ищет первое визуальное вложение в порядке урока: основное фото, основное YouTube-видео, затем дополнительные фото и YouTube-видео. Если подходящего превью нет, используется стандартная обложка.
- Существующие уроки с сохранённой обложкой мигрируются в `custom`, остальные — в `default`.
- Белые углы стандартного изображения скрываются только у стандартных обложек, чтобы собственные изображения не обрезались дополнительно.

## Данные и API

В `content_items` добавляется обязательное поле `cover_mode` со значением `default`, `custom` или `first_material`. Поле проходит через общие схемы, административный API и клиентский API. Сервер валидирует режим и сохраняет его при создании и редактировании урока.

## Интерфейс

В редакторе урока появляется компактная группа из трёх кнопок. Активный вариант визуально выделен. Загрузка и удаление файла доступны только для «Своя обложка». При выборе другого режима сохранённый файл не удаляется автоматически: он остаётся доступен при возврате к режиму `custom`; отдельная кнопка удаления продолжает удалять файл явно.

## Проверка

- Схемы принимают только три режима.
- API сериализует и сохраняет режим.
- Вычисление обложки соблюдает приоритет и fallback.
- Редактор показывает правильные элементы для выбранного режима.
- Стандартная обложка не показывает белые углы на карточке.
- Проверяются мобильная ширина, отсутствие горизонтального переполнения и английские подписи.

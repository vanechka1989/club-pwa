# Редизайн вкладки «Клиенты»

## Цель

Сделать список клиентов быстрым для просмотра, а отдельный экран клиента — компактным рабочим местом администратора без смешивания оперативных действий и длинной истории.

## Список клиентов

- Вверху показывается компактная сводка: всего, с доступом, с ограничениями.
- Поиск остаётся постоянно видимым.
- Основные фильтры остаются быстрыми чипами, остальные — в раскрывающемся блоке.
- Строка клиента содержит аватар, имя, ID, тариф, состояние доступа, прогресс и стрелку перехода.
- В списке нет кнопок управления: нажатие всегда открывает отдельный экран клиента.

## Экран клиента

- Стандартная шапка экрана содержит имя и кнопку возврата; повторяющийся заголовок внутри контента удаляется.
- Первая компактная карточка показывает аватар, ID, последний вход и статусы доступа/тарифа.
- Далее идут четыре одинаковых показателя: доступ до даты, обучение, сумма оплат, последнее действие.
- Кнопка «Написать» доступна рядом с основной информацией.
- Управление доступом вынесено в самостоятельный блок: открыть, закрыть, +7, +30 и ручная дата.
- Профиль, устройство, IP, активность, подписки, оплаты, рефералы и ограничения остаются свёрнутыми разделами.
- Опасные и вторичные действия визуально отделены от основных.

## Адаптивность

- Основная ширина — мобильные экраны 320–430 px.
- Карточки показателей располагаются сеткой 2 × 2, на очень узком экране — одной колонкой.
- Все интерактивные элементы имеют область нажатия не менее 44 px.
- Горизонтальная прокрутка допускается только для быстрых фильтров.

## Ограничения

- API и бизнес-логика доступа, оплат, сообщений и ограничений не меняются.
- Используются существующие темы и семантические CSS-переменные.
- Сохраняется текущая маршрутизация `/admin/clients/:telegramId`.

# Реферальная программа

## Цель

Добавить в клуб реферальную систему: пользователь получает личную ссылку, приглашает нового клиента, а после первой успешной оплаты приглашенного получает бонусные дни доступа. Дни копятся на балансе и активируются вручную.

## Правила

- Реферальная награда начисляется только за первую успешную оплату приглашенного клиента.
- Размер награды задается в настройках проекта, по умолчанию 7 дней.
- Накопленные дни не применяются автоматически.
- Пользователь может активировать бонусные дни только вручную из профиля.
- Активировать бонусные дни нельзя, если у пользователя активная рекуррентная подписка.
- Активация разрешена, если доступ закрыт, истек или был получен разовым/ручным способом.
- Самоприглашение запрещено.
- Реферер закрепляется за приглашенным один раз и не перезаписывается.
- За одного приглашенного клиента нельзя начислить награду повторно.
- Пригласивший получает уведомление, когда новый клиент впервые зашел по его ссылке.
- Пригласивший получает уведомление, когда приглашенный впервые оплатил и бонусные дни начислены.

## Пользовательский сценарий

1. В профиле появляется блок "Реферальная система".
2. Пользователь видит личную ссылку на бота/мини-приложение с идентификатором.
3. Пользователь копирует или отправляет ссылку другому человеку.
4. Новый клиент открывает клуб по ссылке.
5. Система сохраняет связь: кто пригласил и кто пришел.
6. Пригласивший получает уведомление, что по его ссылке зашел новый клиент.
7. После первой успешной оплаты приглашенного система начисляет пригласившему бонусные дни.
8. Пригласивший получает уведомление о платеже и начисленной награде.
9. В профиле пригласившего увеличивается баланс бонусных дней.
10. Пользователь нажимает "Активировать дни", когда это разрешено.

## Данные

Нужны новые сущности:

- `referral_codes`: стабильный код пользователя для ссылки.
- `referrals`: связь пригласившего и приглашенного, дата входа, дата первой оплаты, статус награды.
- `referral_rewards`: начисленные бонусные дни, статус доступно/активировано, дата начисления и дата активации.

Настройка проекта:

- `referral_reward_days`: количество дней за одного оплатившего реферала.

## Backend

### Захват реферала

При авторизации через Telegram initData сервер должен читать стартовый параметр `ref_*`. Если пользователь новый и код принадлежит другому пользователю, создается запись `referrals`.

Если пользователь уже существовал или у него уже есть реферер, новая ссылка игнорируется.

После создания новой реферальной связи нужно создать in-app уведомление пригласившему. Если бот доступен, дополнительно отправить Telegram push через бота.

### Начисление

В обработчике успешной оплаты после выдачи доступа:

- проверить, есть ли у оплатившего реферер;
- проверить, была ли уже награда за этого приглашенного;
- проверить, что это первая успешная оплата приглашенного;
- создать награду на текущий размер `referral_reward_days`;
- отметить реферал как оплаченный;
- создать уведомление пригласившему.

Начисление должно быть идемпотентным, чтобы повторный webhook оплаты не создал дубль.

### Активация

Новый endpoint профиля активирует доступные бонусные дни:

- запрещает активацию при активной рекуррентной подписке;
- суммирует все доступные бонусные дни;
- создает/продлевает подписку с provider `referral_bonus`;
- переводит награды в статус активированных.

Если доступ активен и не рекуррентный, срок продлевается от текущей даты окончания. Если доступа нет или он истек, срок считается от текущего времени.

### Уведомления

Нужны два события:

- "По вашей реферальной ссылке зашел новый клиент".
- "Приглашенный клиент оплатил доступ. Начислено N бонусных дней".

Оба события должны попадать в in-app уведомления. Telegram push отправлять через бота, если пользователь не блокировал бота.

## Frontend

### Профиль

Добавить блок "Реферальная система":

- личная ссылка;
- кнопка копирования;
- доступные бонусные дни;
- количество приглашенных;
- количество оплативших;
- кнопка "Активировать дни".

Если активация запрещена из-за рекуррентной подписки, показать понятный текст: "Бонусные дни можно активировать после окончания или отмены автоплатежа".

### Админка: карточка клиента

В карточке клиента добавить раздел "Рефералы":

- кого клиент пригласил;
- дата и время перехода по ссылке;
- статус: ожидает оплаты / оплачено / награда начислена / награда активирована;
- количество начисленных дней.

Также показать, кто пригласил самого клиента, если есть.

### Админка: Настройки проекта

Добавить новую вкладку "Настройки проекта".

Доступ:

- владелец/разработчик видит всегда;
- обычный админ видит только при праве `project_settings`.

Пока внутри одно поле:

- "Реферальное вознаграждение" в днях.

Сохранение должно валидировать положительное целое число.

## Права

Добавить новое право администратора:

- `project_settings` — "Настройки проекта".

Это право можно выдавать в разделе админов так же, как остальные права.

## Тесты

Покрыть:

- создание реферальной связи только для нового пользователя;
- запрет самоприглашения;
- невозможность перезаписать реферера;
- начисление только за первую успешную оплату;
- идемпотентность webhook;
- уведомление при первом входе по реферальной ссылке;
- уведомление при первой оплате приглашенного;
- запрет активации при активной рекуррентной подписке;
- продление доступа бонусными днями для разового/ручного/истекшего доступа;
- видимость вкладки "Настройки проекта" только владельцу или админу с правом;
- отображение рефералов в карточке клиента.

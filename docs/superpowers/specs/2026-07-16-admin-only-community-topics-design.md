# Скрытые чаты администраторов — дизайн

## Цель

Разрешить владельцу клуба и администраторам с правом «Общение» создавать темы, которые полностью недоступны обычным клиентам.

## Выбранный подход

В таблицу тем добавляется серверный признак `is_admin_only` со значением `false` по умолчанию. При создании темы администратор может включить режим «Только для администраторов». API возвращает признак в модели темы, чтобы интерфейс мог показать замок и понятную подпись.

Это предпочтительнее отдельного типа чатов или отдельной таблицы: тема сохраняет все существующие возможности — сообщения, медиа, опросы, закрепления, архив и realtime — без дублирования логики.

## Права и защита данных

- Доступ имеют владелец и администратор с разрешением `community`.
- Администратор без разрешения `community` рассматривается текущей системой как обычный участник и скрытую тему не получает.
- Обычные участники не получают скрытые темы в обоих списках тем.
- Прямые запросы сообщений, отправка текста, голоса, изображений и опросов возвращают `404`, чтобы не раскрывать существование темы.
- Голосование в опросе и установка реакции также проверяют доступ к теме.
- События realtime скрытой темы не отправляются обычным участникам.
- Архивирование и восстановление продолжают работать по существующим правилам, но не отменяют признак приватности.

## Интерфейс

- В компактной форме создания темы появляется переключатель с замком «Только для администраторов».
- После успешного создания переключатель сбрасывается.
- В списке скрытая тема получает компактную метку с замком «Только админы».
- В открытом скрытом чате под названием показывается «Чат только для администраторов».
- Элементы используют существующие семантические цвета, размеры и минимальную область нажатия 44 px; новые тексты добавляются на русском и английском.

## Ошибки и совместимость

- Старые темы и новые темы без параметра остаются публичными.
- Неверный тип признака отклоняется схемой запроса.
- При отсутствии доступа API отвечает так же, как для несуществующей темы.
- Миграция безопасна для работающей базы: добавляет обязательный boolean с `DEFAULT false`.

## Проверка

- Unit-тесты правила доступа для `member`, `admin`, `owner`.
- Контрактные тесты схемы общей модели.
- Route-тесты/проверки охвата всех клиентских операций с темой.
- UI-тесты формы, метки, заголовка и компактных стилей.
- Полные `test`, `check`, `build`, затем мобильная визуальная проверка списка тем.

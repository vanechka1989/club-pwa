# Предпросмотр контента дополнительных материалов урока

## Проблема

Редактор получает `body` и `existingMediaUrl` каждого дополнительного материала, но показывает только поля формы и подпись «Текущий файл сохранён». Из-за этого сохранённые фото, видео и аудио визуально выглядят отсутствующими. Запись голоса доступна только для основного файла урока, а не для дополнительных аудиоматериалов.

## Решение

- Каждый дополнительный материал показывает текущий контент непосредственно в своей карточке.
- Текст остаётся полностью доступен в textarea.
- Фото отображается изображением, видео — встроенным или YouTube-плеером, аудио — плеером.
- Новый локальный файл или внешняя ссылка сразу заменяет сохранённый источник только в предпросмотре; до сохранения серверные данные не меняются.
- Для аудиоматериала добавляется запись голоса по сценарию чата: старт, таймер и волна, остановка, прослушивание, удаление и применение записи к материалу.
- Одновременно активна только одна запись. Закрытие редактора и удаление материала освобождают микрофон и object URL.

## Ошибки и мобильное поведение

- Ошибка разрешения микрофона показывается рядом с соответствующим материалом.
- Контент не выходит за ширину карточки на 320 px; медиаплееры занимают 100% доступной ширины.
- Все управляющие кнопки имеют минимум 44 × 44 px и доступные названия.

## Проверка

- Unit-тест выбора источника предпросмотра для всех типов контента.
- Source/component-тест наличия предпросмотра и голосового сценария в дополнительных карточках.
- Typecheck, полный набор тестов, production build.
- Playwright-аудит редактора урока на Android 320/390, iPhone/WebKit и планшете.

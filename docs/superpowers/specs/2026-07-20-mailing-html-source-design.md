# HTML-код в рассылках

## Цель

Позволить администратору вставлять безопасную HTML-разметку в текст рассылки так, чтобы письмо и уведомление внутри приложения отображались форматированными, а системный PWA push содержал читаемый текст без HTML-тегов.

## Интерфейс редактора

В форме создания рассылки рядом с панелью форматирования появляется переключатель «Визуально / HTML-код». В визуальном режиме сохраняется текущий `contenteditable`-редактор. В режиме HTML-кода показывается многострочное поле с исходной разметкой. Переключение синхронизирует оба представления без потери поддерживаемых тегов.

HTML-режим принимает вставку вида `<b>текст</b>` как разметку, а не как видимые клиенту символы. Предпросмотр использует тот же очищенный HTML, который будет сохранён и отправлен.

## Безопасная разметка

Сервер остаётся источником истины и очищает HTML библиотекой `sanitize-html`. Разрешены: `p`, `br`, `strong`, `b`, `em`, `i`, `u`, `code`, `ul`, `ol`, `li`, `a`, `blockquote`, `h2`, `h3`. Для ссылок разрешены только `http`, `https` и `mailto`; они открываются в новой вкладке с `noopener noreferrer`. Скрипты, стили, обработчики событий, изображения и прочие теги или атрибуты удаляются.

Клиентский санитайзер центра уведомлений должен разрешать тот же набор отображаемых тегов, включая `code`, и повторно блокировать опасные атрибуты.

## Доставка по каналам

- Email получает очищенный HTML и отображает форматирование.
- Центр уведомлений приложения получает `bodyHtml` и отображает его через клиентский санитайзер.
- Нативный PWA push получает производный plain-text: теги удалены, переносы строк и маркеры списков сохранены. HTML никогда не попадает в системное уведомление.

Если HTML пуст после очистки, отправка блокируется существующей проверкой обязательного текста. Текстовая версия формируется на сервере из очищенного HTML.

## Проверка

Тесты должны подтвердить поддержку `b`, `i` и `code`, удаление опасной разметки, корректное преобразование примера в plain-text для push, синхронизацию двух режимов редактора и отсутствие HTML-тегов в payload системного push. Затем выполняются полный набор тестов и production-сборка.

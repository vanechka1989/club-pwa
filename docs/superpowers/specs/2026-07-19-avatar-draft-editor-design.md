# Черновик фото профиля

## Цель

Исправить экран изменения фото профиля: аккуратно разместить карточку над нижней навигацией, показывать текущее фото, сразу показывать выбранный локальный файл и не изменять сохранённую аватарку до нажатия «Сохранить».

## Интерфейс

- Меню фото остаётся компактной нижней карточкой, но располагается выше нижней навигации и безопасной зоны.
- В карточке показывается круглый предпросмотр текущей аватарки с сохранённым кадрированием.
- Действия «Загрузить новое фото» и «Настроить кадр» сохраняются.
- После выбора файла открывается редактор кадра с локальным предпросмотром выбранного изображения.
- Редактор текущего фото использует сохранённый URL и параметры кадрирования.
- «Отмена» закрывает редактор, освобождает локальный URL и ничего не отправляет на сервер.
- «Сохранить» применяет выбранное фото и кадрирование. Во время сохранения кнопки блокируются и показывается состояние загрузки.

## Данные и атомарность

Компонент хранит отдельный черновик: выбранный `File`, локальный object URL, позицию X/Y и масштаб. Состояние `session.user` не меняется при выборе файла. Для нового фото клиент отправляет файл и параметры кадрирования одним запросом; сервер записывает ссылку и кадрирование одним обновлением пользователя. Для действующего фото остаётся отдельный PATCH кадрирования.

Если запрос не прошёл, сохранённая аватарка в клиентском состоянии не заменяется, редактор остаётся открыт, а пользователь видит ошибку. Object URL отзывается при отмене, успешном сохранении и размонтировании компонента.

## Проверки

- Модульные тесты проверяют разбор параметров кадрирования на API и отправку формы клиентом.
- Тесты профиля проверяют независимость черновика от текущего пользователя и наличие предпросмотра в меню.
- Playwright на мобильном viewport проверяет карточку над навигацией, текущее фото, локальный предпросмотр, отмену без запроса и применение только после сохранения.

# Фильтры клиентов по источнику и UTM

## Цель

Добавить в раздел «Клиенты» совместимые с существующими фильтрами отборы по источнику регистрации и UTM-меткам. Фильтрация должна работать мгновенно на уже загруженном списке, не создавать отдельный запрос на каждого клиента и сохранять аккуратную мобильную компоновку.

## Выбранный подход

Сводка `/admin/stats` получает для каждого клиента компактное поле `acquisition` с последним источником перед регистрацией: `source`, `medium`, `campaign`, `content`. Сервер загружает эти данные одним запросом для всех показанных клиентов. Это соответствует карточке клиента, где источником регистрации уже считается последнее касание.

Клиентская фильтрация остаётся локальной и объединяется по логическому AND с поиском, доступом, тарифом и ограничениями. Такой вариант быстрее и проще текущего интерфейса, чем серверная пагинация, и не допускает N+1-запросов.

## Интерфейс

В раскрывающемся блоке «Дополнительные фильтры» появятся:

- «Любой источник» — точный выбор из реально присутствующих `utm_source`, включая «Без метки»;
- «Любая UTM-метка» — выбор поля `utm_source`, `utm_medium`, `utm_campaign` или `utm_content`;
- поле «Значение UTM» — регистронезависимый поиск по выбранному полю или сразу по всем полям.

Контролы используют текущую адаптивную сетку, минимальную высоту 44 px и не добавляются в горизонтальную строку быстрых фильтров. Сброс очищает и новые параметры. Список источников и значений строится без дублей и сортируется по алфавиту.

## Данные и ошибки

Если у клиента нет атрибуции, `acquisition` равен `null`; он попадает в вариант «Без метки». Отсутствие отдельных UTM-полей не считается ошибкой. Старые ответы API остаются совместимыми благодаря необязательному полю схемы.

## Проверка

- тест контракта общей схемы;
- тест чистой функции комбинированной фильтрации, регистра без учёта регистра и «Без метки»;
- тест наличия серверного пакетного запроса и UI-контролов;
- полный `check`, unit-тесты и production-сборка;
- маршрутный аудит Android 320 px и iPhone/WebKit без горизонтального переполнения.

# Подтверждение удаления сообщений в чате

## Цель

Заменить системный браузерный `window.confirm` при массовом удалении сообщений чата на тематизированный диалог внутри PWA.

## Интерфейс

- Переиспользуется общий `ConfirmDialog`, уже применяемый в оплате и поддержке.
- Заголовок: «Удалить все сообщения?».
- Для владельца описание сообщает, что удаление произойдёт сразу и без восстановления.
- Для остальных модераторов описание сообщает о скрытии сообщений и окончательной очистке через 24 часа.
- Основное действие называется «Удалить всё», имеет опасный статус и красное семантическое оформление.
- Вторичное действие называется «Отмена».
- Меню администратора закрывается до открытия диалога.
- Во время запроса обе кнопки заблокированы; повторный запрос невозможен.

## Поведение

- Нажатие «Отмена» закрывает только диалог.
- Нажатие на фон равнозначно отмене, если удаление ещё не выполняется.
- После успешного удаления закрываются состояния модерации, перечитывается текущая тема и обновляется список тем.
- При ошибке диалог закрывается, а существующий механизм уведомлений показывает ошибку сообщества.
- Бизнес-логика API и правила немедленного/отложенного удаления не меняются.

## Доступность и темы

- Диалог имеет `role="alertdialog"`, `aria-modal="true"` и доступное имя.
- Минимальная высота кнопок сохраняется не менее 44 px.
- Используются существующие семантические токены поверхностей, текста и опасного действия, поэтому диалог работает во всех темах.

## Проверка

- Автоматический тест запрещает `window.confirm` в массовом удалении темы и проверяет подключение `ConfirmDialog`.
- Проверяются подписи, опасное состояние и busy-состояние.
- Запускаются тесты сообщества, полная проверка типов и production-сборка.

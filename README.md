# Шаблон PWA-клуба

Шаблон платного закрытого клуба как самостоятельное PWA-приложение. Вход выполняется по email-коду, уведомления идут через встроенный центр уведомлений и браузерные PWA push-уведомления.

- Backend: Bun, Hono, TypeScript strict, Zod, Drizzle ORM, PostgreSQL, pino.
- Frontend: Vue 3, Vite, Pinia, ofetch, Tailwind CSS 4.
- Общий пакет: Zod-схемы и TypeScript-типы для API и frontend.
- План развития: профиль, обучение, оплата, поддержка и минимальная цель 1000 зарегистрированных пользователей.

Пока не добавлены в стартовый шаблон: Redis, BullMQ, S3/Garage, `jose`, Argon2 и rate limiter. Их стоит подключать тогда, когда появятся очереди, загрузка файлов, отдельные web-сессии, пароли или публичные endpoints с риском злоупотреблений.

## Быстрый Старт

Нужно установить: `pnpm`, `bun` и Docker.

```bash
pnpm install
cp .env.example .env
docker compose up -d postgres
pnpm db:generate
pnpm db:migrate
pnpm dev
```

Web-приложение откроется на `http://localhost:5173`.

План проекта хранится в [PROJECT_PLAN.md](./PROJECT_PLAN.md).
Текущая архитектура описана в [ARCHITECTURE.md](./ARCHITECTURE.md).

## Локальный Email-Вход

Для разработки в обычном браузере API может показывать код входа в ответе `/auth/email/start`:

```env
OWNER_EMAIL=owner@example.com
ADMIN_EMAILS=admin@example.com
AUTH_DEV_CODE_ENABLED=true
```

В production оставьте `AUTH_DEV_CODE_ENABLED=false` и настройте SMTP, иначе коды входа не смогут отправляться пользователям:

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=club@example.com
SMTP_PASSWORD=replace-me
SMTP_FROM=club@example.com
```

Seed-данные для категорий и первых материалов:

```bash
pnpm db:seed
```

Команда требует запущенный PostgreSQL и применённые миграции.

## Установка На Сервер

Основной сценарий для нового клиента: зайти на чистый VPS по SSH и вставить одну команду. Она не требует токена GitHub: сервер скачает готовые Docker-образы шаблонного клуба, создаст настройки, применит миграции, выпустит HTTPS-сертификат через Caddy и запустит приложение.

```bash
curl -fsSL https://club.myn8nservertest.ru/install-club.sh | bash
```

Установщик спросит:

- папку установки;
- домен клиента;
- адрес клуба;
- email владельца;
- дополнительные email админов;
- SMTP для кодов входа;
- добавлять ли базовый демо-контент.

Web Push VAPID ключи установщик создаёт автоматически и сохраняет в `.env`. При повторном запуске существующие ключи сохраняются, чтобы не сбросить уже выданные браузером подписки.

Важно: Docker-образы должны быть публичными в реестре контейнеров GitHub:

- `ghcr.io/vanechka1989/club-pwa-api`
- `ghcr.io/vanechka1989/club-pwa-web`

После первого запуска GitHub Actions `Публикация образов шаблонного клуба` нужно один раз сделать эти образы публичными в GitHub: `Repository -> Packages -> package -> Package settings -> Change visibility -> Public`.

Для PWA нужен HTTPS-домен. API публикуется на том же домене по пути `/api`:

```text
Web: https://club.example.com
API: https://club.example.com/api
Health: https://club.example.com/api/health
```

После установки обновление на сервере:

```bash
bash /opt/club-pwa/update.sh
```

## Админка

Админка открывается внутри того же PWA. Главный админ задаётся переменной `OWNER_EMAIL`. Нижняя вкладка `Админ` появляется у владельца и у пользователей, добавленных в админы.

Как проверить или изменить владельца:

1. Указать email, которым владелец входит в клуб.
2. На сервере открыть `/opt/club-pwa/.env`.
3. Проверить строку:

```env
OWNER_EMAIL=owner@example.com
```

Дополнительных админов можно добавлять и удалять прямо во вкладке `Админ`. Владелец не удаляется из вебки, чтобы случайно не потерять доступ.

Для ручного добавления через `.env` всё ещё можно использовать:

```env
ADMIN_EMAILS=admin1@example.com,admin2@example.com
```

После изменения `.env` перезапустить сервисы:

```bash
cd /opt/club-pwa
docker compose -f docker-compose.prod.yml up -d
```

После этого открыть PWA заново. В нижнем меню появится вкладка `Админ`.

У владельца есть режим просмотра mini app:

- `Как есть` - реальное состояние аккаунта.
- `Без доступа` - посмотреть клуб глазами пользователя без подписки.
- `Купил доступ` - посмотреть клуб глазами пользователя с активной подпиской.

Этот режим не меняет реальные подписки и работает только для `OWNER_EMAIL`.

## Интерфейс

Frontend поддерживает русский и английский язык, переключение темы день/ночь, установку как PWA и нижнее меню для мобильного браузера. Выбранный язык и тема сохраняются в браузере пользователя.

## Прогресс Уроков

Приложение запоминает последний открытый урок пользователя и позволяет отметить материал как пройденный. В разделе обучения показывается прогресс: сколько материалов пройдено и какой урок был открыт последним.

## Управление Доступом

Во вкладке `Админ` можно смотреть статистику клиентов, искать клиента по email, видеть прогресс уроков и вручную менять доступ: выдать, забрать или выставить дату окончания подписки.

Дополнительный сценарий: установка с локальной машины на сервер по SSH.

```bash
read -rs GITHUB_TOKEN
bash <(curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" https://raw.githubusercontent.com/vanechka1989/club-pwa/main/deploy/install.sh)
```

## Поток PWA

Frontend запрашивает код на `/auth/email/start`, затем подтверждает его через `/auth/email/verify`.
API создаёт cookie-сессию `club_session`, привязывает пользователя к email и возвращает статус подписки через `/me`.

PWA push-уведомления работают после автогенерации `WEB_PUSH_PUBLIC_KEY` и `WEB_PUSH_PRIVATE_KEY` установщиком, регистрации service worker и согласия пользователя в браузере.
